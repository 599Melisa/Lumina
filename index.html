Mission

Why we built Lumina

Three reasons we decided to build our own model from scratch instead of using existing solutions.

01

Freedom without compromises

Every major AI model today is restricted by filters. ChatGPT refuses 23% of requests, Claude — 18%. We believe the user should decide what to ask. Lumina has no built-in censorship — neither during training nor inference. No RLHF restrictions, no preset "red lines".

02

Power on weak hardware

Most good models require 24-80 GB VRAM. We optimized Lumina to run on consumer GPUs. INT4 quantization, PagedAttention, and a compact 30B parameter architecture allow the model to run on GPUs with just 6 GB of memory — with generation speed comparable to top cloud solutions (58 tokens/sec).

03

Open training library

We don't just release weights — we provide a full toolkit. The lumina-lib library lets you fine-tune the model on your data, change its behavior, and adapt it to tasks — all locally, without sending data to any server.

lumina-lib

Training library

Full toolkit for fine-tuning, customization, and interaction with Lumina. Everything runs locally.

Generation

Simple interface for sending prompts and receiving responses. Supports streaming, system prompts, temperature settings.

Fine-tuning

Upload your dataset in JSONL format — the library automatically prepares data, configures LoRA adapters, and starts training.

Quality evaluation

Built-in benchmarks for measuring quality before and after fine-tuning. MMLU, HumanEval, custom tests on your data.

Quick start

# Install
pip install lumina-lib

# Load model
from lumina import LuminaModel

model = LuminaModel.load("lumina-30b-int4")

# Generate response
response = model.generate(
  prompt="Explain quantum entanglement",
  max_tokens=512,
  temperature=0.7
)
print(response.text)

Fine-tuning on your data

# Prepare data (data.jsonl)
# {"instruction": "...", "output": "..."}

from lumina import LuminaTrainer

trainer = LuminaTrainer(
  base_model="lumina-30b-int4",
  data_path="data.jsonl",
  lora_rank=16,
  learning_rate=2e-4,
  epochs=3
)

trainer.train()
trainer.save("my-lumina-adapter")

# Use fine-tuned model
model = LuminaModel.load("lumina-30b-int4", adapter="my-lumina-adapter")

Streaming responses

for chunk in model.stream("Write a poem about neural networks"):
  print(chunk.text, end="", flush=True)

FAQ

Frequently asked questions

Can I use Lumina in commercial projects?

Yes. Model weights are distributed under the Apache 2.0 license — free commercial use, modification, and distribution. The lumina-lib library is also open source.

What are the minimum requirements to run it?

The INT4 version (30B) requires a minimum of 6 GB VRAM — that's a GPU like GTX 1660, RTX 2060 or better. For comfortable use we recommend 8+ GB. The full-precision version requires ~60 GB VRAM.

Why is the quality lower than ChatGPT or Claude?

We deliberately sacrificed some quality for the absence of censorship. The RLHF process that competitors use to improve quality simultaneously introduces strict filters. We abandoned RLHF — this is our conscious choice. Quality can be improved through fine-tuning with lumina-lib.

Can I fine-tune the model for a specific task?

Yes, that's exactly what lumina-lib was created for. You prepare a dataset in JSONL format (instruction and output fields), run LoRA fine-tuning — and get an adapter that can be connected to the base model. Training runs locally, data is never sent anywhere.

Is my data sent to your servers?

No. When running locally, all data is processed on your device. Lumina has no telemetry, doesn't send requests, and doesn't log anything. We don't even know how many people use the model.

How is Lumina different from other uncensored models?

Existing uncensored models are usually forks of other models with removed filters. Lumina was trained from scratch without censorship at all stages. This provides more consistent behavior without artifacts that arise when "cutting out" filters from someone else's model. Plus we provide a training library.

Are context size and language count planned to increase?

Yes. The roadmap for v2.1 includes expanding context to 128K and adding 20+ more languages. Follow updates in our repository.
